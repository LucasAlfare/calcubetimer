All of the text in CCT is formatted as dynamic strings. Dynamic strings can be used as plaintext, but can also access statistics that CCT keeps track of. Using dynamic strings, you can get CCT to display just the information that you are interested in.

Here's a simple example: "Your best time: $$besttime$$". If your best time is 15.12 seconds, this will be formatted by CCT as "Your best time: 15.12". However, to make it easy to translate your xml layout, you can externalize the "Your best time" string as well. The dynamic string would then become "%%besttimemessage%%: $$besttime$$. If your layout is called "My Awesome Layout.xml", then you must also create "My Awesome Layout.properties" containing this line: "besttimemessage=Your best time". NOTE: This will not work if your xml layout file contains more than one period. See CCT Readme for a more detailed description of properties files and internationalization.

What follows is a list of all the $$ properties accessible within CCT. As described above, to distinguish the keys from plaintext, surround them with $$ and $$, for example: $$pops$$. To print "$$pops$$" as plaintext, just use "\$\$pops\$\$" Some keys may require a numeric argument specifying which RA you are interested in. Remember that CCT keeps track of all the statistics for two rolling average lengths. For example, to get the latest average for a given RA number, use $$currentAverage0$$ or $$currentAverage1$$. For any key ending in #, the # should be replaced with the number of the desired RA.

NOTE: Standard deviations for trimmed (rolling) averages are computed over only the solves included in the average.
FURTHER NOTE: Capitalization of the $$ keywords does not matter.

$$ Keyword Description
sessionAverage This will be replaced with the average of all non infinite solves in the times list (current session).
sessionSD This is the standard deviation of all non infinite solves in the current session.
pops Number of pops in the current session.
+twos Number of 2 second penalties in the current session.
dnfs Number of DNFs in the current session.
solves Number of successful (non-infinite) solves in the current session.
attempts Total number of attempts in in the current session.
progressTime The difference between the most recent solve and the solve before it.
progressAverage# The difference between the most recent RA # and the previous RA #.
progressTimeParens This formats progressAverage between parenthesis, and will be an empty string if the difference was infinite.
progressAverageParens# see progressTimeParens
bestTime This is the best time in the current session, or N/A if there are no solves.
bestRA# The is the best rolling average of RA #.
bestAverageSD# This is the standard deviation of the best average of RA #.
bestSD# The is the best standard deviation of RA #. Not necessarily the same as bestAverageSD, for example, you could have a slower average with very little variation.
worstTime This is the worst, finite time in the current session.
worstAverage# This is the worst average of RA #.
worstAverageSD# The standard deviation of the worst average in RA #.
worstSD# This is the worst standard deviation of all rolling averages in RA #.
currentTime The most recent time.
currentAverage# The current average of RA #.
currentSD# The standard deviation of the current average of RA #.
lastTime This is the time before the currentTime.
lastAverage# This is the 2nd most recent average of RA #.
lastSD This is the standard deviation of the average in lastAverage#.
bestTimeOfCurrentAverage# Straightforward.
worstTimeOfCurrentAverage# Straightforward.
bestTimeOfBestAverage# Straightforward.
worstTimeOfBestAverage# Straightforward.
bestTimeOfWorstAverage# Straightforward.
worstTimeOfWorstAverage# Straightforward.
progressSessionAverage This is the difference between the current session average and the one before it.
progressSessionSD This is the difference between the SD of the current session and the one before it.
progressSessionAverageParens This formats progressAverageSD nicely.
progressSessionSDParens This formats progressSessionSD nicely.
bestAverageList# A comma separated list of the times in the best average of RA #. The best and worst times are surrounded with parenthesis.
currentAverageList# see bestAverageList#
sessionAverageList see bestAverageList#
worstAverageList# see bestAverageList#
bestAverageStats# This enumerates the times of the best average of RA #, and appends their scramble and comment.
currentAverageStats# see bestAverageStats#
sessionStats see bestAverageStats#
bestAverageStatsWithSplits# This enumerates the times of the best average of RA #, and appends their scramble, splits, and comment.
currentAverageStatsWithSplits# see bestAverageStatsWithSplits#
sessionStatsWithSplits see bestAverageStatsWithSplits#
The following allow access to global statistics. That is, statistics about every solve you have ever done of the current scramble customization.
veryBestTime This is the best time of all the sessions of the current scramble customization.
veryBestRA# The best RA of RA # of all the sessions of the current scramble customization.
globalAverage The average of all solves of all the sessions of the current scramble customization.
globalPops The total number of pops of all the sessions of the current scramble customization.
global+twos The total number of 2 second penalties of all the sessions of the current scramble customization.
globalDNFs The total number of DNFs of all the sessions of the current scramble customization.
globalSolveCount The total number of successful solves of all the sessions of the current scramble customization.
globalAttemptCount The total number of solve attempts of all the sessions of the current scramble customization.
Misc
date This is the current date and time. This will not be updated every second.
